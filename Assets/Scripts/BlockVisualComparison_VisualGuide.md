## 1. 육각형 블록 ASCII 아트 비교

### 1.1 Red 블록 - Before (현재) / After (개선) 비교

```
┌─────────────────────────────────────────────────────────────────┐
│                       RED 블록 비교                              │
└─────────────────────────────────────────────────────────────────┘

[BEFORE - 현재 상태]
평평한 2D 원판 느낌, 밝고 단조로운 명암

        배경(거의 흰색)               젬(밝은 빨강)           테두리(약함)

          ╔═══════════╗                ╭──────────╮          ┏━━━━━┓
         ╱   ░░░░░░░░░░╲              ╱  OOOO...  ╲         ┃░░░░░┃
        ╱     ░░░░░░░░░░  ╲          ╱   OOO○○OOO   ╲        ┃░░░░░┃
       │      ░░░░░░░░░░░   │        │  OO■■■■■OO   │       ┃░░░░░┃
        ╲     ░░░░░░░░░░  ╱          ╲   OOO○○OOO   ╱        ┃░░░░░┃
         ╲   ░░░░░░░░░░╱              ╲  OOOO...  ╱         ┗━━━━━┛
          ╚═══════════╝                ╰──────────╯

색상값:                               색상값:                 색상값:
RGB(0.96, 0.93, 0.90) ×              RGB(0.95, 0.15, 0.15)   RGB(0.92, 0.88, 0.85)
α=0.28 (투명 크림색)                 α=0.95 (밝음)          α=0.30 (약함)
명도: 0.93                            명도: 0.52 (균일함)    명도: 0.88

문제점 분석:
▢ 배경과 젬의 명도 차이 작음 (0.93 vs 0.52)
▢ 그래디언트 부재: 중심-가장자리 명암 변화 미세
▢ 테두리가 약해 윤곽 구분 어려움
▢ 평면적: 입체감/쿠션감 거의 없음


[AFTER - 개선 상태]
입체적 쿠션 느낌, 중심 밝음 → 가장자리 어두움 그래디언트

        배경(움푹 들어감)              젬(구체형)              테두리(강함)

          ╔═══════════╗                ╭──────────╮          ◆━━━━━◆
         ╱   ░░░░░░░░░░╲              ╱  ✱✱✱✱✱✱  ╲         ◇━━━━━◇
        ╱   ░░░░░▒▒░░░░  ╲          ╱   ✱✱✱✱✱✱✱✱   ╲        ◇▒▒▒▒▒◇
       │    ░░▒▒▒▒▒▒░░░   │        │  ◦✱■■■■■✱◦   │       ◇▒▒▒▒▒◇
        ╲   ░░░░░▒▒░░░░  ╱          ╲   ✱✱✱✱✱✱✱✱   ╱        ◇━━━━━◇
         ╲   ░░░░░░░░░░╱              ╰──────────╯         ◆━━━━━◆
          ╚═══════════╝                ▲ 좌상단
                                       하이라이트
                                       (백색 0.3α)
색상값:                               색상값:                 색상값:
RGB(0.65~0.95) 그래디언트            RGB(0.95, 0.15, 0.15)   RGB(0.94, 0.91, 0.88)
중심: 0.85, 가장자리: 0.70           중심: 0.70              α=0.45 (강화)
명도: 동심원 그래디언트              명도: 0.70-0.95         명도: 0.91 (베벨 강함)

개선 효과:
✓ 배경 그래디언트: 움푹 들어간 슬롯 느낌 강화
✓ 젬 입체감: 구체처럼 중심-가장자리 명암 차이
✓ 하이라이트 레이어: 좌상단 빛 반사 표현 (α 0.3~0.5)
✓ 테두리 강화: 명확한 윤곽 + 외부 글로우
✓ 깊이감: 다층 구조 (배경-젬-하이라이트-테두리)
```

### 2.2 테두리 강화: 베벨 강도 시각화

```
┌─────────────────────────────────────────────────────────────────┐
│              테두리(Border Ring) 강화 - 베벨 비교                 │
└─────────────────────────────────────────────────────────────────┘

[BEFORE - 약한 베벨]

육각형 테두리 프로필 (위쪽 모서리 기준):

명도      ╱╲  ← 베벨 링 (10px)
0.95 ────╱  ╲────────  가장자리 밝음
         │ ▓▓▓ │
0.88 ────┤ ▓▓▓ ├────  베벨 어두운 쪽 (거의 차이 없음)
         │     │
0.80     │     │

베벨 강도: 0.95 - 0.88 = 0.07 (매우 약함)
테두리 알파: 0.30 (거의 투명)


[AFTER - 강화된 베벨]

명도      ╱╲╲  ← 강화 베벨 (10px) + 외부 글로우
1.00 ────╱  ╲╲──────  외부 글로우 (0.95, 0.3α)
         │ ◇◇◇◇ │
0.95 ────┤ ◇◇◇◇ ├─────  가장자리 밝음
         │ ◆◆◆◆ │
0.85 ────┤       ├─────  베벨 밝은 쪽
         │       │
0.70 ────┤       ├─────  베벨 어두운 쪽

베벨 강도: 0.95 - 0.70 = 0.25 (3배 강화!)
테두리 알파: 0.45 (명확)
외부 글로우: 2~3px, 부드러운 확산


[비교표]

구성요소         Before          After           개선도
─────────────────────────────────────────────────────
베벨 밝은 쪽    0.92            0.94            +0.02
베벨 어두운 쪽  0.88            0.70            -0.18 (어두워짐)
베벨 강도       0.04            0.24            6배 강화
테두리 알파     0.30            0.45            1.5배 명확
외부 글로우     ~8%             2~3px           추가됨
선택/강조       유지            유지 (더욱 돋보임)


[단면도 - 위에서 본 테두리 링]

Before:
육각형 ┌─────────┐
      │ ░░░░░░░ │  (약한 테두리, 약간의 베벨만)
      │ ░XXXXXXX │
      │ ░X     X │
      │ ░XXXXXXX │
      │ ░░░░░░░ │
      └─────────┘

After:
육각형 ╔═════════╗◈  (강한 테두리 + 외부 글로우)
      ║ ◇◇◇◇◇◇◇ ║◈
      ║ ◇XXXXXXX ║
      ║ ◇X     X ║
      ║ ◇XXXXXXX ║
      ║ ◇◇◇◇◇◇◇ ║◈
      ╚═════════╝◈

◈ = 외부 글로우 (2~3px)
```

단점: 약간 더 복잡한 렌더링


[프로시저럴 하이라이트 스프라이트 함수 개념]

로직:

2. 각 픽셀 (x, y)에 대해:

     * 거리 가까움: 강도 높음 (1.0)
     * 거리 멈: 강도 낮음 (선형 또는 가우시안 감쇠)

3. 스프라이트로 반환

의사 코드:
```
for each pixel (x, y):
    if not inside hexagon:
        alpha = 0
    else:
        distToUpperLeft = sqrt((x-10)² + (y-10)²)
        intensity = exp(-distToUpperLeft / 20)  // 가우시안 감쇠

4. 최종 오프셋:
   - 하이라이트: +0.20 × intensity
   - 음영: -0.15 × (1 - intensity)

## 7. 참고 자료 및 관련 파일

### 주요 파일 경로
```
Assets/Scripts/Core/HexBlock.cs
  ├─ CreateAAHexSprite()        → 배경/테두리 프로시저럴
  ├─ CreateAAInnerHexSprite()   → 젬 프로시저럴

### 기술 개념
- **SDF (Signed Distance Field)**: 정확한 육각형 경계 표현
- **Perlin Noise**: 마카롱 미세 질감
- **동심원 그래디언트**: 중심에서 가장자리로 부드러운 명도 변화
- **방향성 조명 (Directional Lighting)**: 좌상단 기준 일관된 광원
- **가우시안 감쇠 (Gaussian Falloff)**: 자연스러운 페이드 아웃
- **에지 어두움 (Rim Darkening)**: 경계 부분의 그림자 강조
