## 🎮 레벨 디자인 원칙

### 기본 철학
1. **점진적 학습**: 각 스테이지마다 1개의 색상도둑부터 시작해 최대 3개까지 증가
2. **명확한 피드백**: 색상도둑으로 인한 문제와 해결책이 명확해야 함
3. **플레이어 안내**: 오라클리온의 대사와 UI 가이드로 적극적 지시
4. **성공 경험**: 처음부터 끝까지 "나는 할 수 있다"는 느낌 제공

### 난이도 곡선
```
Stage 1-2: 색상도둑 1개 (적군 소개 + 드릴 학습)
Stage 3-5: 색상도둑 2개 (복잡도 증가)
Stage 6-8: 색상도둑 2-3개 (보드 배치 다양화)
Stage 9-10: 색상도둑 3개 + 추가 난이도 (클리어 테스트)
```

## 🗺️ 보드 레이아웃 디자인

### 좌표계 설명
- **그리드 크기**: 반지름 5 육각형 (약 61개 블록)
- **중심**: (0, 0)
- **좌표 시스템**: Axial Coordinates (q, r)
  - q: 좌우 축 (음수 = 좌, 양수 = 우)
  - r: 위아래 축 (음수 = 위, 양수 = 아래)

### 육각형 그리드 구조
```
반지름 5인 육각형 (1~5 확장 레이어):

        (-2,-3)(-1,-3)(0,-3)(1,-3)(2,-3)
      (-3,-2)(-2,-2)(-1,-2)(0,-2)(1,-2)(2,-2)
    (-4,-1)(-3,-1)(-2,-1)(-1,-1)(0,-1)(1,-1)(2,-1)
  (-5,0)(-4,0)(-3,0)(-2,0)(-1,0)(0,0)(1,0)(2,0)(3,0)
    (-4,1)(-3,1)(-2,1)(-1,1)(0,1)(1,1)(2,1)
      (-3,2)(-2,2)(-1,2)(0,2)(1,2)(2,2)
        (-2,3)(-1,3)(0,3)(1,3)(2,3)

(중심이 (0,0), 총 61개 블록)
```

## 📋 Stage별 레벨 디자인

### **Stage 1: 적군 소개 - 색상도둑 1개**

**목표**: 색상도둑 1개 제거
**턴 제한**: 25턴
**난이도**: ⭐☆☆☆☆ (매우 쉬움)

#### 스토리 배경
```
[컷씬]
루나가 크리스탈 숲에서 처음 보석 블록들을 만난다.
오라클리온: "자, 이게 보석의 격자야. 같은 색 3개가 모이면 정화된단다."
루나가 회전을 배우며 몇 번 성공한다.

[새로운 문제 등장]
갑자기 한 블록이 회색으로 변한다.
색상도둑이 나타난다!
오라클리온: "어라? 이건 무언가 이상한데... 색상 도둑이로군!"
프리즘: "어? 보석의 색이 사라졌어!"
루나: "어떻게 해야 하는데요?"
오라클리온: "특수 보석을 써보거라. 드릴 같은 특수 기술이면..."
```

#### 보드 배치

```
기본 배치:
- 색상: Red, Blue, Green, Yellow, Purple 5종류
- 색상도둑 위치: (0, 0) - 중앙에 1개만 배치
- 나머지: 자연스러운 배경 색상

ASCII 맵:
      [ ][ ][ ]
    [ ][ ][ ][ ]
  [ ][ ][ ][도][ ][ ]
[ ][ ][ ][ ][ ][ ][ ]
  [ ][ ][ ][ ][ ][ ]
    [ ][ ][ ][ ]
      [ ][ ][ ]

[도] = 색상도둑 (회색 표시)
```

#### 특수 블록 배치
- **드릴 생성 가능 영역**: 보드 전체 (4매칭 이상)
- **보너스**: 드릴 생성 난이도 낮음 (3.5매칭 이상 생성 가능)
- **의도**: 플레이어가 자연스럽게 드릴을 시도해볼 수 있도록

#### 플레이 가이드 (UI)

**시작 전**:
```
[튜토리얼 팝업]
제목: "색상도둑이 나타났어요!"
내용:
- "보석의 색이 사라졌어요!"
- "그레이 색상의 블록은 일반 매칭에 사용할 수 없어요."
- "특수 블록(드릴, 폭탄 등)을 사용하면 제거할 수 있어요!"

[스킵 버튼] [확인 버튼]
```

**게임 중**:
- 색상도둑 블록에 붉은색 테두리 강조
- 색상도둑이 있는 블록 위에 "회색 슬라임" 아이콘 표시
- "색상도둑 제거: 0/1" 진행도 바 (좌상단)

**오라클리온 가이드**:
```
[게임 시작]
오라클리온: "저 회색 블록이 색상 도둑이야. 일반 매칭으로는 안 되지."

[플레이어가 색상도둑에 인접한 블록을 매칭]
오라클리온: "흠... 색상도둑은 여전히 남아 있군."

[플레이어가 드릴 생성]
오라클리온: "오! 드릴이 생겼어! 색상도둑을 향해 발사해봐!"

[드릴로 색상도둑 제거]
오라클리온: "좋아! 색상도둑이 정화되었구나!"
루나: "와, 정말 효과가 있네요!"

[미션 완료]
오라클리온: "축하한다. 이제 넌 색상도둑의 정체를 알게 되었네."
```

#### 게임플레이 팁
1. **적극적 유도**: 드릴 생성이 쉬워서 플레이어가 자연스럽게 시도
2. **명확한 성공**: 드릴로 제거하면 즉시 시각적 피드백 (파워풀한 이펙트)
3. **턴 여유**: 25턴은 여유 있게 설계 (보통 10-15턴 소요)
4. **재도전 불필요**: 첫 시도에서 80% 이상이 성공

#### 예상 플레이 흐름
```
1. 게임 시작 - 오라클리온 튜토리얼
2. 보드 관찰 - 색상도둑 발견
3. 몇 번 회전 시도 - 색상도둑이 사라지지 않음을 깨달음
4. 드릴 생성 (자연스럽게 발생)
5. 드릴 사용 - 색상도둑 제거
6. 미션 완료 - 축하 애니메이션
```

### **Stage 2: 복습 - 색상도둑 1개 + 새로운 배치**

**목표**: 색상도둑 1개 제거
**턴 제한**: 28턴
**난이도**: ⭐☆☆☆☆ (쉬움)

#### 스토리 배경
```
[컷씬]
루나가 숲을 더 깊이 들어간다.
프리즘: "루나, 이번엔 어디에 색상도둑이 있을까?"
루나: "조심히 찾아봐야겠네."

[보드 진입]
오라클리온: "이번엔 색상도둑의 위치가 다르군. 같은 원리지만 자리가 달라."
```

#### 보드 배치
- **색상도둑 위치**: (2, -2) - 중앙 우측 상단
- **의도**: 보드의 다양한 위치에서 처리할 수 있음을 경험
- **난이도**: Stage 1과 동일하지만, 위치 변경으로 약간의 신선함 제공

```
      [ ][ ][ ]
    [ ][ ][ ][ ]
  [ ][ ][ ][  ][ ][도]
[ ][ ][ ][ ][ ][ ][ ]
  [ ][ ][ ][ ][ ][ ]
    [ ][ ][ ][ ]
      [ ][ ][ ]

[도] = 색상도둑 (우측 상단)
```

#### 특수 블록 배치
- 드릴 생성 난이도 정상화 (4매칭 이상)
- 폭탄 생성 가능 (5매칭 이상)
- **의도**: 드릴 또는 폭탄으로 자유로운 선택

#### 플레이 가이드
- 색상도둑 위치 맵에 표시 (화살표)
- "자, 이번엔 위치가 달라. 같은 방법으로 해결할 수 있을까?"
- 더 이상 강한 튜토리얼 팝업 X (스킵한 플레이어 배려)

#### 예상 플레이 흐름
```
1. 색상도둑 위치 발견
2. 드릴 또는 폭탄 생성
3. 사용하여 제거
4. 미션 완료
```

### **Stage 3: 난이도 상승 - 색상도둑 2개**

**목표**: 색상도둑 2개 제거
**턴 제한**: 30턴
**난이도**: ⭐☆☆☆☆ (쉬움, 복잡도 증가)

#### 스토리 배경
```
[컷씬]
루나가 진행하던 중 색상도둑이 2마리 나타난다.
프리즘: "어? 2개나? 너무 많은데!"
루나: "하나씩 해결하면 되겠지!"
오라클리온: "그렇지. 하지만 색상도둑이 많을수록 보드가 더 복잡해진단다."
```

#### 보드 배치

**색상도둑 위치**:
- 색상도둑 1: (-2, 1) - 좌측 아래
- 색상도둑 2: (1, -2) - 우측 위
- **의도**: 대칭 배치로 균형 잡힌 난이도

```
      [ ][ ][도]
    [ ][ ][ ][ ]
  [ ][ ][ ][ ][ ][ ]
[ ][ ][ ][ ][ ][ ][ ]
  [ ][도][ ][ ][ ][ ]
    [ ][ ][ ][ ]
      [ ][ ][ ]

[도] = 색상도둑 (2개)
```

#### 특수 블록 배치
- 드릴: 자연 생성 가능
- 폭탄: 자연 생성 가능
- **보너스**: 조금 더 빈번한 특수 블록 생성

#### 플레이 가이드

**UI**:
```
"색상도둑 제거: 0/2" 진행도 바

색상도둑 1 제거 시:
"색상도둑 제거: 1/2" → 하나 남음을 시각화

색상도둑 2 제거 시:
"색상도둑 제거: 2/2" → 완료!
```

**오라클리온**:
```
게임 시작: "이번엔 색상도둑이 2개로군. 하나씩 차근차근히 처리해봐."

첫 번째 제거 후: "좋아! 하나 남았군. 저쪽도 정화해봐."

두 번째 제거 후: "완벽하군! 두 색상도둑 모두 정화했어!"
```

#### 플레이 전략
1. **우선순위 결정**: 어떤 색상도둑부터 제거할지
2. **특수 블록 활용**: 2개 모두 처리 가능할 특수 블록 선택
   - 폭탄(범위 좋음) vs 드릴(지정 좋음)
3. **연쇄 활용**: 한 색상도둑 제거 후 자동 낙하로 다음 특수 블록 생성

### **Stage 4: 레이아웃 다양화 - 색상도둑 2개 + 장애물**

**목표**: 색상도둑 2개 제거
**턴 제한**: 32턴
**난이도**: ⭐⭐☆☆☆ (약간 어려움)

#### 스토리 배경
```
[컷씬]
루나가 더 위험한 지역으로 진입한다.
오라클리온: "이 숲 깊숙이로 들어갈수록 오염이 심해지는군."
프리즘: "앗, 색상도둑 말고 다른 것도 있어?"
루나: "그래도 이겨낼 수 있어!"
```

#### 보드 배치

**특징**:
- 색상도둑 2개 + 고정 블록(FixedBlock) 3-4개 추가
- 고정 블록은 매칭 불가, 이동 불가 (장애물)
- 색상도둑 위치가 더 떨어져 있음

**색상도둑 위치**:
- 색상도둑 1: (-3, 0) - 좌측
- 색상도둑 2: (3, -1) - 우측

```
      [ ][ ][ ]
    [ ][ ][X][ ]
  [ ][ ][ ][ ][도][ ]
[ ][도][ ][ ][ ][ ][ ]
  [ ][X][ ][ ][ ][ ]
    [ ][ ][ ][X]
      [ ][ ][ ]

[도] = 색상도둑
[X] = 고정 블록 (장애물)
```

#### 난이도 분석
- **색상도둑의 거리 증가**: 한 번에 2개 제거 어려움 → 차례대로 처리 필수
- **고정 블록의 영향**:
  - 일부 특수 블록 경로 차단
  - 낙하 흐름 변경
  - 전략적 사고 필요

#### 플레이 전략
1. **거리 관리**: 먼 위치의 색상도둑 먼저 또는 나중에?
2. **장애물 우회**: 특수 블록 경로 계획
3. **연쇄 활용**: 고정 블록이 연쇄를 막을 수 있으므로 주의

#### 게임플레이 팁
```
오라클리온: "이번엔 고정 블록들이 길을 막고 있군."
"하지만 색상도둑을 하나씩 처리하면 된다."
```

### **Stage 5: 복합 장애물 - 색상도둑 2개 + 다양한 조건**

**목표**: 색상도둑 2개 제거
**턴 제한**: 35턴
**난이도**: ⭐⭐☆☆☆ (약간 어려움)

#### 스토리 배경
```
[컷씬]
루나: "색상도둑이 자꾸만 나타나네요."
오라클리온: "오염이 점점 심해지고 있는 것 같다. 조심스럽게 진행하거라."
프리즘: "루나는 할 수 있어! 이겨내!"
```

#### 보드 배치

**특징**:
- 색상도둑 2개 + 고정 블록 4-5개
- 보드 전체적으로 더 복잡한 배치
- 특수 블록 생성이 다소 어려움 (더 높은 매칭 필요)

**색상도둑 위치**:
- 색상도둑 1: (-1, -2)
- 색상도둑 2: (2, 2)

```
      [ ][ ][도]
    [ ][X][ ][ ]
  [ ][ ][ ][ ][ ][ ]
[X][ ][ ][ ][ ][ ][ ]
  [ ][ ][ ][ ][ ][도]
    [ ][ ][X][ ]
      [ ][X][ ]

[도] = 색상도둑 (2개, 거리 멀음)
[X] = 고정 블록 (장애물 많음)
```

#### 난이도 분석
- **위치 분산**: 색상도둑이 보드 끝과 끝
- **이동 제약**: 고정 블록이 중앙부를 차단
- **장기전**: 35턴 시간 압박 X, 하지만 전략 필수

#### 플레이 전략
```
1. 중앙 정리: 고정 블록 주변 자원 활용
2. 색상도둑 처리: 가까운 쪽부터 또는 먼 쪽부터?
3. 특수 블록 확보: 충분한 매칭으로 드릴/폭탄 생성
4. 연쇄 활용: 낙하와 특수 블록 조합
```
