# 기본 블록 매트 마감 + 입체감 개선 사양서 v1.0

**작성일**: 2026-02-17
**담당**: 아트 디렉터
**대상**: 기본 젬 블록(Red, Blue, Green, Yellow, Purple, Orange) 렌더링 시스템 개선

---

## 1. 현재 블록 시각적 문제점 분석

### 현재 렌더링 방식
- **배경(Background)**: UI.Image에 그레이스케일 육각형 → 색상 틴팅 적용
  - 색상: `(0.96f, 0.93f, 0.90f, 0.28f)` (약간 투명한 크림색)
  - 문제: 매우 밝고 평평한 느낌, 입체감 없음

- **젬(GemImage)**: UI.Image에 프로시저럴 또는 외부 텍스처
  - 프로시저럴: `CreateAAInnerHexSprite()`로 마카롱 텍스처 생성 (미세 질감)
  - 셰이더: `UI/HexGem` 또는 `UI/HexSpecialGem` 적용 (깊이/하이라이트/SSS)
  - 문제: 셰이더의 깊이 표현이 완전하지 않음, 질감만으로는 입체감 부족

- **테두리(Border)**: 베벨 테두리 링, 외부 글로우 거의 없음
  - 색상: `(0.92f, 0.88f, 0.85f, 0.30f)` (밝은 회색)
  - 문제: 테두리가 약하고 블록 경계 강조 부족

### 시각적 문제점
1. **평평한 느낌**: 2D 이미지처럼 보임, 큰 깊이감 부족
2. **매트한 표면이지만 생동감 부족**: 광택 없는 것은 맞지만 입체 음영 부족
3. **색상 명도 불균형**: 밝은 색상이 블록을 "떨어뜨려" 보임
4. **그라디언트 부재**: 중심-가장자리의 명암 변화가 미세함

## 2. 매트 마감 + 입체감 개선 방안

### 2.1 디자인 방향

| 항목 | 현재 | 개선 후 | 효과 |
|------|------|--------|------|
| **마감질감** | 매트 (기존) | 매트 유지 + 미세 울퉁불퉁 | 광택 없으면서 구조감 |
| **입체감** | 음영 미약 | 중심 밝음 → 가장자리 어두움 | 볼록/쿠션 느낌 |
| **테두리** | 약한 베벨 | 강화 베벨 + 외부 글로우 | 블록 윤곽 강조 |
| **깊이감** | 거의 없음 | 다층 구조: 배경-젬-하이라이트| 입체적 표현 |
| **색감** | 고르고 밝음 | 명암 + 색온도 변화 | 자연스러운 3D 느낌 |

### 2.2 구현 아이디어

#### A. 그래디언트 + 음영 처리 (프로시저럴 개선)

**목표**: 프로시저럴로 생성되는 젬 텍스처에 동심원 그래디언트 추가

```
프로시저럴 젬 텍스처 (CreateAAInnerHexSprite 개선 버전):

1. 기본 구조 유지:
   - 육각형 경계: SDF로 정확한 모양
   - 마카롱 질감: PerlinNoise (기존, 0.97~1.0)

2. 추가 요소:
   - 동심원 그래디언트: 중심 밝음 → 가장자리 어두움
     * 중심: +0.1 (밝음)
     * 가장자리: -0.15 (어두움)

   - 방향성 조명 하이라이트: 좌상단에서 빛
     * 주 하이라이트: 좌상단 1/3 영역, +0.2 밝기
     * 보조 하이라이트: 우하단 1/4 영역, +0.08 밝기

   - 에지 어두움 (Rim Darkening): 가장자리 1~2픽셀
     * 테두리 근처: -0.2 어두움
     * 자연스러운 경계 강조

3. 색상 처리:
   - 기본색: 틴팅 색상 유지
   - 음영 추가: 다크닝/밝기 조정 후 색상 적용
```

**구현 난이도**: 중간 (기존 함수 개선)

---

#### B. 하이라이트 레이어 시스템 (새 UI 요소)

**목표**: 젬 위에 별도의 하이라이트 이미지 오버레이 추가

```
계층 구조:
  HexBlock (루트)
    ├─ BackgroundImage (슬롯 배경)
    ├─ GemImage (젬 - 프로시저럴 또는 외부 텍스처)
    ├─ HighlightImage (NEW - 하이라이트 레이어)
    ├─ BorderImage (테두리)
    └─ OverlayImage (기타 효과)

HighlightImage:
- 스프라이트: 프로시저럴 하이라이트 패턴
- 위치: 좌상단 1/3 영역에 집중
- 색상: 흰색 (0.95, 0.95, 1.0) × 0.3~0.5 알파
- 블렌딩: 가산 또는 스크린 모드
- 크기: GemImage와 동일
- raycastTarget: false
```

**프로시저럴 하이라이트 패턴**:
```
CreateHighlightSprite(int size):
  - 육각형 내부, 좌상단에 집중
  - 중심: 밝음 (백색)
  - 가장자리: 페이드 아웃
  - 방향성: 좌상단 벡터 방향으로 이동
  - 부드러운 Soft Light 패턴
```

**구현 난이도**: 낮음~중간 (UI 구조 추가, 새 스프라이트 함수)

---

#### C. 배경 음영 강화 (슬롯 배경)

**목표**: 배경 이미지에 더 강한 음영 그래디언트 추가

```
현재:
  - 밝은 베벨: 0.78 ~ 0.88 범위 (매우 좁음)
  - 외부 글로우: 거의 없음

개선:
  - 베벨 범위 확대: 0.65 ~ 0.95 범위 (더 명암 차이)
  - 중심부 어두움: 0.70 (기존보다 -0.08)
  - 가장자리 밝음: 0.95 (유지)
  - 안쪽 움푹 들어간 느낌 강화
```

**구현 위치**: `HexBlock.cs` → `CreateAAHexSprite()` 개선

**구현 난이도**: 낮음 (상수 조정)

---

#### D. 테두리 강화 (Border Ring)

**목표**: 테두리를 더 두드러지게 하여 블록 윤곽 강조

```
현재:
  - 베벨 링: 10px × 방향성 라이팅
  - 색상: (0.92, 0.88, 0.85) × 0.30
  - 외부 글로우: ~8% 정도

개선:
  1. 베벨 밝기 증가:
     - 밝은 쪽: 0.88 ~ 0.95
     - 어두운 쪽: 0.60 ~ 0.70

  2. 외부 글로우 추가:
     - 너비: 2~3px
     - 색상: (0.95, 0.92, 0.90) × 0.4
     - 페이드: 부드러운 확산

  3. 색상 강화:
     - 현재보다 +0.05 ~ +0.08 밝게
     - 선택/강조 시 더욱 강화
```

**구현 위치**: `HexBlock.cs` → `CreateAAHexSprite()` isBorderOnly 섹션

**구현 난이도**: 낮음 (상수 조정)

### 2.3 VFX 통합 (Phase 1 패턴 유지)

기존 Phase 1 VFX 패턴과의 호환성:

```
✓ Pre-Fire Compression (0.78 → 1.18 스케일): 변경 없음
✓ Hit Stop + Slomo: 변경 없음
✓ Zoom Punch (hexGrid.transform 스케일): 변경 없음
✓ Destroy Flash (백색 오버레이): 블록 파괴 시 여전히 강력하게 표현됨
✓ Dual Easing Destroy (20% 확대 → 80% 스퀴즈): 변경 없음
✓ Bloom Layer (센터 플래시 + 글로우): 배경 개선로 더욱 강조
✓ Match Pulse (매칭 스케일 펄스): 입체감으로 펄스가 더 돋보임

신규 상호작용:
- 새로운 HighlightImage 계층은 Destroy Flash와 조화
  : 파괴 시 하이라이트도 함께 백색으로 치환 (선택적)
- 입체감 강화로 전체 VFX 임팩트 증가
```

## 5. 성능 고려사항

### 메모리
- 새 HighlightImage 캐싱: +0.3MB (한 번만)
- 프로시저럴 개선: 변화 없음 (기존 크기)
- 전체 증가: 무시할 수준

### CPU
- Phase 1 (상수 조정): 영향 없음
- Phase 2 (프로시저럴 강화): 스타트업에만 +5~10ms (무시할 수준)
- 런타임: 변화 없음 (캐싱됨)

### 권장사항
- 모든 개선은 게임플레이 성능에 영향 없음
- 안전하게 적용 가능

## 7. 예상 결과

### Before (현재)
```
- 평평한 2D 원판처럼 보임
- 색상만으로 구분, 입체감 부족
- 테두리가 약함
- 블록이 "떨어진" 느낌
```

### After (개선 후)
```
- 입체적인 쿠션/젤리 같은 볼록함
- 중심 밝음 → 가장자리 어둠으로 깊이감
- 테두리가 명확하게 블록 윤곽 강조
- 블록이 "튀어나온" 느낌
- 여전히 매트한 마감 (광택 없음)
- VFX 임팩트 증가
```

## 8. 참고 자료

- `HexBlock.cs`: 블록 렌더링 메인

  - `GetGemSprite()`, `GetBorderSprite()`

### 기술 참고
- **Signed Distance Field (SDF)**: 정확한 육각형 경계
- **Perlin Noise**: 마카롱 미세 질감
