## 미션 관리 도구

$ARGUMENTS

모든 응답은 한국어로 작성합니다.

---

### 모드 선택

인자에 따라 다음 모드 중 하나를 실행합니다:

- **분석** (기본): 지정 범위의 미션 구성을 테이블로 출력
- **검증**: 미션 데이터의 문제점을 자동 감지
- **수정**: 감지된 문제를 자동 수정

---

### 분석 모드

**파일**: `Assets/Scripts/Data/Mission1StageData.cs`

지정 범위의 모든 스테이지를 읽고 다음 테이블을 출력합니다:

| 스테이지 | 턴 | 난이도 | 미션 수 | 활성제한 | 미션 구성 | 총 몬스터 | 비고 |
|---------|---|-------|--------|---------|----------|----------|------|
| 1 | 12 | 1 | 3 | 1 | G2→G2→G3 | 7 | 순차웨이브 |
| 2 | 13 | 1 | 3 | 1 | G3→G2→G3 | 8 | 순차웨이브 |

**표기법:**
- `G`=몽둥이, `A`=갑옷, `R`=궁수, `S`=방패, `B`=폭탄, `H`=힐러, `V`=헤비, `W`=마법사, `T`=도둑, `Wi`=마녀
- `G2`=Lv2 몽둥이, `A2`=Lv2 갑옷, `R2`=Lv2 궁수, `S2`=Lv2 방패
- `→` = 순차(maxActive로 대기열 사용), `,` = 동시 활성

### 검증 모드

다음 규칙을 자동으로 검사합니다:

#### 규칙 1: 동일 타입 동시 활성 중복 금지
- 미션 수 ≤ maxActiveMissions(기본4)이고, 같은 EnemyType이 2회 이상 → **경고**
- 수정 방법: 합산하거나 maxActive 설정으로 순차 소환

#### 규칙 2: 연속 스테이지 동일 구성 금지
- 인접 스테이지(N, N+1)의 미션 구성이 동일한 타입 조합 + 유사한 수량(±1) → **경고**
- 판정: 타입 집합이 동일하고, 각 타입의 수량 차이 합이 2 이하

#### 규칙 3: 단일 타입 스테이지는 순차 소환
- 미션이 1종류 타입만 사용하고 미션 수 ≥ 2 → maxActive=1 필수
- 미설정 시 **경고**

#### 규칙 4: 미션-턴 밸런스
- 총 몬스터 수 / 턴 제한 > 1.0 → **경고** (턴당 1마리 이상 처치 필요 = 난이도 높음)
- 총 몬스터 수 / 턴 제한 < 0.3 → **경고** (너무 쉬움)

#### 규칙 5: 점진적 난이도
- 같은 챕터 내에서 총 몬스터 수가 이전 스테이지보다 크게 감소(3 이상) → **경고**

출력 형식:
```
[경고] Stage 5-6: 연속 동일 구성 (G만 단독, 총수량 유사)
[경고] Stage 14: 동일 타입 동시 활성 (A×2, G×2)
[통과] Stage 21-30: 규칙 위반 없음
```

### 수정 모드

검증에서 발견된 문제를 자동으로 수정합니다:

1. **동일 타입 동시 활성 중복** → 합산
2. **단일 타입 순차 소환 미설정** → maxActive=1 추가
3. **연속 동일 구성** → 수량/웨이브 구조 변경 제안 (자동 수정은 위험하므로 제안만)

수정 후 반드시 `dotnet build "Assembly-CSharp.csproj"` 컴파일 검증을 수행합니다.

### 참고: 미션 시스템 구조

**StageData.maxActiveMissions**: 동시 활성 미션 수 제한 (0=기본 MAX_ACTIVE_MISSIONS=4)

**미션 큐 시스템** (StageManager.cs):
- 미션 배열의 처음 maxActive개 → 즉시 활성화
- 나머지 → pendingMissions 대기열
- 활성 미션 완료 시 → PromotePendingMissions()로 다음 미션 승격
- 동일 타입 미션이 동시 활성 → 킬 시 첫 번째 미완료 미션만 카운트 (break)

**설계 원칙**:
- 같은 타입은 동시 활성 X → 합산하거나 순차(maxActive) 사용
- 단일 타입(챕터1 등)은 웨이브 수/수량에 변화를 줘서 다양성 확보
- 연속 스테이지는 구조(웨이브 수, 순서, 변칙)를 달리해야 함
- 새 타입 첫 등장 스테이지는 해당 타입 소량 + 기존 타입으로 부드러운 도입
