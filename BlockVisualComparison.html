2. VFX 개선 항목 (Phase별)

주의: 모든 개선사항은 UI.Image 기반 프로시저럴 생성입니다. ParticleSystem이나 외부 에셋을 사용하지 않습니다.

Phase 1: 핵심 Juice (3~4일)

I-01: Hit Stop (순간 정지)

특수 블록 폭발 순간, 게임이 1~3프레임 멈추고 슬로모로 진행되는 효과

효과: 임팩트 체감 200% 증가

구현: Time.timeScale 조작 (WaitForSecondsRealtime 사용)

높음

I-02: 화이트 플래시

블록 파괴 순간 순백색 오버레이가 번쩍하고 사라지는 효과

효과: 파괴의 "시원한" 피드백

성능: 블록당 1 GO, 0.06초 수명

높음

P-01: 특수블록 충전 효과

발동 전 0.1초 동안 블록이 안쪽으로 수축했다가 팽창하는 "숨" 효과

적용: Drill, Bomb, Laser, Donut, XBlock

스케일: 1.0 → 0.85 → 1.12

높음

P-02: 매칭 하이라이트 펄스

매칭 확인 직후 블록이 1.08배로 팽창했다가 수축하는 효과

효과: 매칭 인지도 200% 증가

추가: 흰색 반투명 오버레이 + 스태거

높음

M-01: 이중 이징 파괴

파괴 애니메이션을 20% 팽창 → 80% squeeze+shrink로 분리

패턴: Anticipation + Action (디즈니 12원칙)

기간: 0.14초 (기존 유지)

높음

S-02: 줌 펀치

특수 블록 폭발 시 보드가 1.02x로 줌인했다가 원래 크기로 복귀

지속시간: 0.03s 줌인 + 0.08s 아웃

효과: 화면이 튀어나오는 느낌

높음

C-01: 블룸 시뮬레이션

밝은 이펙트 뒤에 큰 글로우 레이어를 깔아 "빛이 번지는" 효과

방식: 플래시의 1.8배 크기, 0.25배 알파

성능: 이펙트당 +1 GO

높음

Phase 2: 폴리시 (3~4일)

F-01: 고스트 이미지 잔상

블록 파괴 후 0.15초 동안 해당 위치에 색상 잔상이 남았다가 사라짐

스케일: 0.8 → 1.2 (확산하며 사라짐)

알파: 0.35 → 0

중간

F-02: 낙하 착지 더스트

블록이 낙하하여 도착할 때 하단에서 먼지가 퍼지는 효과

개수: 좌우 1개씩 (2개)

방향: 위쪽 30px/s + 좌우 확산

중간

A-01: 특수블록 호흡 애니메이션

보드 위의 특수 블록이 천천히 스케일 오르락내리락하고 빛남

속도: 2.5Hz 호흡 + 3.0Hz 글로우

스케일: 0.97 ↔ 1.03

중간

S-01: 방향성 셰이크

특수 블록별로 셰이크 방향을 다양화

Drill: 수직 흔들림

Laser: 3축 교대 흔들림

중간

M-02: 스파크 꼬리 (Elongation)

스파크가 이동 방향으로 길쭉해지며 "꼬리를 끌며" 날아가는 효과

비율: 길이 2x + 폭 0.6x

비용: 연산만 (GO 추가 없음)

중간

M-03: 스프링 이징

EaseOutElastic보다 짧은 진동으로 빠르게 안정되는 이징

주기: p=0.15 (0.3의 절반)

용도: 충격파 복귀, 블록 바운스

중간

C-03: HSV 색상 다양성

파편/스파크 색상을 HSV 공간에서 더 다양하게 변환

Hue: ±15° | Saturation: ±0.15 | Brightness: ±0.2

효과: 색감 풍부도 대폭 증가

중간

Phase 3: 프리미엄 터치 (2~3일)

P-03: 콤보 빌드업 에스컬레이션

캐스케이드 깊이에 따라 보드 경계 테두리와 보드 줌이 증가

Depth 2: 테두리 alpha 0.1 | Depth 3: + 줌 1.015 | Depth 4+: alpha 0.3 + 줌 1.03

중간

I-03: 디스토션 링 (블록 밀기)

폭탄/레이저 폭발 시 주변 블록이 바깥으로 밀렸다 돌아오는 물리적 반응

반경: 200px | 밀이: 최대 5px

복귀: EaseOutSpring, 0.15s

중간

I-04: 드릴 트레일 잔상

드릴 투사체 이동 경로에 0.2초 동안 남는 반투명 라인

너비: 4px | 알파: 0.5

성능: 타겟당 1 GO

중간

F-03: 여진 웨이브

폭탄/레이저 폭발 완료 후 0.3초 뒤에 작은 충격파 1회 추가

크기: 기존의 60% | 알파: 기존의 50%

효과: 마무리감 및 "끝났다" 신호

중간

C-02: 색온도 시프트

특수 블록 발동 시 보드 전체 색감이 해당 시스템 테마로 틴팅

Bomb: 따뜻한 톤 | Laser: 차가운 톤 | Rainbow: 무지개 순환

중간

S-03: 보드 미세 바운스

매칭 확인 시 보드 전체가 1.5px 아래로 내려갔다 올라오는 미세 반동

다운: 0.03s | 업: EaseOutElastic, 0.08s

효과: 모든 매칭에 무게감 부여

중간

최적화 팁

GO 생성 최소화:

각 이펙트는 필요한 최소 개수만 생성하고, lifetime을 명확히 설정하여 자동 소멸 처리

TimeScale 관리:

Hit Stop 적용 시 cooldown 0.3s를 두어 연속 호출 방지. 항상 WaitForSecondsRealtime 사용

Easing 함수:

새 EaseOutSpring(p=0.15)을 짧은 복귀 애니메이션에 활용. EaseOutElastic은 장시간 애니메이션에만

Transform Hierarchy:

효과는 effectParent의 자식으로 생성. hexGrid.transform은 shake/bounce/zoom만 조작

3가지만 구현해도 되는 최소 필수 조합

1️⃣ I-01: Hit Stop

TimeScale 조작 한 줄로 임팩트 체감 200% 증가

2️⃣ I-02: 화이트 플래시

블록당 Image 1개, 0.06s로 "시원한 파괴감"

3️⃣ M-01: 이중 이징

기존 코드 수정만으로 "팽창+붕괴" 느낌
